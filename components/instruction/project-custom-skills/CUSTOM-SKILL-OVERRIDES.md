# Custom Skill Overrides

No system-wide project override is defined.

An overlay may add evidence-backed paths, versions, conventions, validation commands, or narrowing constraints. It may not:

- weaken the Constitution, Standard Skills, security, compatibility, testing, or production rules;
- change a reusable base in place;
- invent a domain invariant or technology version;
- contain a secret, credential, personal/production data, or raw log;
- expand a binding beyond its detected package/path scope.
