# Patterns

## Progressive retrieval

Use index, then summary, then detail, then source verification. This limits context while preserving traceability.

## Source-backed synthesis

Store a concise claim plus repository-relative sources and evidence. Keep long code, schema, logs, and specifications in their authoritative locations.

## Atomic detail/index update

Create or update the detail entry and its index row in one change. Lint immediately to prevent unrouteable knowledge.

## Scope-nearest with constitutional guardrails

Prefer the closest feature/module memory, but always load applicable project-wide security, production, permission, data-integrity, and public-contract constraints.

## Explicit uncertainty

Use provisional for bounded assumptions, supported for incomplete evidence, and disputed for conflicts. Add the exact verification step rather than inflating confidence.

## Bidirectional supersession

Preserve history and link both old and new entries. Default retrieval excludes the old entry while regression/history analysis can still find it.

## Incident learning loop

Capture symptom, reproduction, root-cause status, failed attempts, final fix, regression protection, prevention, and detection. Do not promote a workaround to the final solution.

## Checkpoint compaction

Keep active working state exact and resumable. On completion, extract durable memory, create one task summary, and archive raw working state.

## Controlled pattern promotion

Record a successful one-off solution as provisional. Promote only after reuse, strong technical evidence, or explicit acceptance.
