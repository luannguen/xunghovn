# Checklists

## Pre-task gate

- [ ] Read AI-BOOTSTRAP and this skill.
- [ ] Classify task and risk.
- [ ] Identify domain, module, feature, entities, affected paths, dependencies, permissions, and integrations.
- [ ] Read index and project constraints.
- [ ] Retrieve closest domain, module, feature, incident, debt, question, and checkpoint entries.
- [ ] Verify important memory against current sources.
- [ ] Detect overdue, stale, missing-source, and conflicting memory.
- [ ] Create a concise Memory Context Summary with source paths.
- [ ] Load the applicable specialist skill.
- [ ] For continuation, verify task ID, branch/version, paths, pending steps, errors, tests, blocker, and exact next action.

Medium/complex implementation is blocked until every applicable item above is complete. Small tasks require a lightweight scan; none may skip memory entirely.

## Post-task gate

- [ ] Identify new material fact, invariant, decision, pattern, failed approach, root cause, incident, debt, open question, module map, API/schema contract, and documentation delta.
- [ ] Keep unverified claims provisional or disputed.
- [ ] Update/supersede stale entries with traceability.
- [ ] Update detail and structured index atomically.
- [ ] Check sources, relationships, status, confidence, dates, scope, and triggers.
- [ ] Confirm no secret, private production data, copied long source, or chain-of-thought.
- [ ] Create an exact checkpoint if unfinished.
- [ ] Create a compact completed summary and close active working memory if finished.
- [ ] Run skill validation when the skill changed.
- [ ] Run memory lint; resolve every BLOCKER.
- [ ] Report Memory Update concisely.

## Incident close gate

- [ ] Symptom, reproduction, environment/version, impact, and affected modules recorded.
- [ ] Root cause status is explicit.
- [ ] Failed attempts and temporary mitigation are distinguished from the permanent fix.
- [ ] Regression test, detection signal, and prevention are recorded.
- [ ] Rollback/recovery and relevant changes are linked.
