# Task Execution Brief

Create this brief before implementation for MEDIUM, HIGH, and CRITICAL tasks. Keep it concise and evidence-based.

## Template

- Task:
- Goal and acceptance criteria:
- Task class:
- Risk level and rationale:
- Scope:
- Explicit exclusions:
- Verified evidence:
- Unknowns and assumptions:
- Selected workflow:
- Selected skills and why:
- Relevant memory IDs:
- Invariants and affected contracts:
- Approval gates:
- Implementation sequence:
- Validation plan:
- Rollback or recovery:
- Documentation and memory plan:
- Blockers and stop conditions:
- Expected deliverables:

For LOW tasks, retain the same decisions internally and report only material risks or gates.
