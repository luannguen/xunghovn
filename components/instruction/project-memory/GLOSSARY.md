# Glossary

- authoritative source: the current source allowed to establish a claim for its information type.
- evidence: a path, test, runtime observation, approved decision, or other reproducible support.
- implementation reality: behavior expressed by current runtime/code/schema.
- intended contract: approved specification, business rule, architecture decision, or protected contract.
- memory entry: indexed synthesis with scope, trigger, source, status, confidence, and lifecycle.
- Memory Context Pack: selected, verified memory used for one task.
- procedural memory: rules and workflows.
- semantic memory: stable project facts, constraints, boundaries, and contracts.
- episodic memory: incidents, migrations, failures, and historical outcomes.
- working memory: resumable task state awaiting completion/compaction.
- stale: requiring re-verification because source, time, dependency, or scope changed.
- disputed: credible sources conflict and authority is unresolved.
- superseded: replaced but retained for traceability.
- trigger term: concept, error, feature, entity, or operation that makes an entry retrievable.
