# Memory Remediation Backlog

## P0 - On source arrival

1. Re-run repository inventory and update MEM-PROJECT-0001.
2. UI/UX Production and Backend Production Engineering are now routed; execute their source-triggered audit backlogs when application sources arrive.
3. Execute audits/UI-UX-REMEDIATION-BACKLOG.md and audits/BACKEND-REMEDIATION-BACKLOG.md when relevant application sources arrive.
4. Read approved product/game specifications and create only source-backed business invariants.
5. Map manifests, entry points, modules, boundaries, shared services/components, and dependency direction.
6. Map schema/migrations, data ownership, authentication, authorization, permission, and public contracts.
7. Locate active/blocked tasks, incidents, failed approaches, migrations, and accepted decisions.

## P1 - Establish enforcement

1. Map and route application-specific coding, architecture, testing, security, documentation, migration, and deployment rules under the existing specialist skills.
2. Record testing architecture, deployment environments, observability, rollback, and incident workflow.
3. Identify frequently changing paths, technical debt, repeated failures, undocumented workarounds, and unstandardized patterns using evidence.
4. Create module memories and integration contracts with review intervals based on volatility/risk.
5. Initialize or restore Git only with project-owner direction.

## P2 - Improve retrieval after evidence

1. Exercise representative frontend, backend, bug, continuation, migration, and documentation queries.
2. Tune tags/triggers and relevance routing based on observed misses.
3. Promote provisional patterns only after reuse or explicit acceptance.
4. Schedule garbage collection and stale review when memory volume warrants it.

## Exit criteria

The baseline onboarding gap closes only when application modules, authoritative business/technical rules, application-specific UI/backend contracts, tests/runtime verification, and deployment context are source-backed and indexed. Do not fabricate entries to make the backlog appear complete.
